## 【现场编码演示：智能体路由】
首先，进入项目目录（cd backend/services/agents），使用ls命令查看文件，可以看到agents文件夹包含了所有智能体代码。
接下来我将**现场编写**一个简化版的多智能体路由系统，展示核心原理。
新建一个demo.py文件，开始编写代码：
```python
# ===== 现场敲这段代码 =====
class Agent:
    """智能体基类"""
    def __init__(self, name, keywords):
        self.name = name
        self.keywords = keywords
    
    def confidence(self, text):
        """计算处理置信度：关键词匹配度"""
        return sum(1 for k in self.keywords if k in text) / len(self.keywords)
# 创建4个专家智能体
agents = [
    Agent("慢病专家", ["血压", "血糖", "高血压", "糖尿病"]),
    Agent("生活教练", ["运动", "睡眠", "饮食", "锻炼"]),
    Agent("心理关怀", ["焦虑", "担心", "心情", "孤独"]),
    Agent("健康管家", ["你好", "查询", "报告"])
]
def route(question):
    """置信度路由：选择最合适的智能体"""
    scores = [(a.name, a.confidence(question)) for a in agents]
    best = max(scores, key=lambda x: x[1])
    print(f"问题: {question}")
    print(f"→ 路由到: {best[0]}，置信度: {best[1]:.0%}\n")
# 测试路由效果
route("我血压有点高怎么办")
route("最近睡眠不太好")  
route("心情有点低落")
```
核心思想是**置信度路由算法**。每个智能体根据自己的专长领域计算处理置信度，协调器选择置信度最高的智能体来响应用户。这避免了简单的关键词匹配，实现了智能化的任务分发。
运行代码（python demo.py），可以看到：
- "血压高"问题自动路由到慢病专家
- "睡眠不好"路由到生活教练  
- "心情低落"路由到心理关怀
至此，我已完成智能体路由核心算法的演示。

## 【RAG知识库演示】
接下来演示RAG知识库的构建和检索。
打开backend/services/knowledge_base.py文件，这是RAG的核心实现：
```python
class KnowledgeBase:
    def __init__(self):
        # 使用m3e-base模型（中文优化）
        self.embedding_model = SentenceTransformer("moka-ai/m3e-base")
        self.embedding_dim = 768
        # FAISS向量索引
        self.index = faiss.IndexFlatIP(self.embedding_dim)
    
    def search(self, query: str, top_k: int = 5):
        """语义检索相关知识"""
        query_vec = self.embedding_model.encode([query])[0]
        scores, indices = self.index.search(query_vec.reshape(1, -1), top_k)
        return [(self.metadata[i], scores[0][j]) for j, i in enumerate(indices[0])]
```
首先，配置嵌入模型为m3e-base，这是专门针对中文优化的向量模型。
接着，初始化FAISS索引，维度为768。
然后，search方法实现语义检索：将用户问题向量化，在知识库中检索最相关的文档。
知识库中已导入高血压防治指南、糖尿病管理规范、中国居民膳食指南等权威医学文献，共计数万条知识条目。
