## 🎯 改进清单

### P0 - 高优先级（必须修复）

#### 1. [ ] 前端密码明文存储问题
- **文件**：`frontend/src/components/login/ElderlyLoginPage.tsx`
- **问题**：密码明文存储到 localStorage
- **违反标准**：GB/T 35273-2020 个人信息安全规范
- **解决方案**：使用 Token 机制替代密码存储，登录成功后仅存储 JWT Token
- **预计工时**：1小时

#### 2. [ ] 添加隐私政策页面
- **文件**：需新建 `frontend/src/components/PrivacyPolicy.tsx`
- **问题**：缺少隐私政策说明页面
- **违反标准**：GB/T 35273-2020 要求明示个人信息收集目的
- **解决方案**：
  - 创建隐私政策页面，说明数据收集范围、用途、存储方式
  - 首次登录时弹出隐私政策确认弹窗
  - 用户勾选同意后方可使用系统
- **预计工时**：2-3小时

#### 3. [ ] 用户数据删除机制
- **文件**：`backend/api/routes/user.py`、`frontend/src/components/MyInfo.tsx`
- **问题**：用户无法请求删除个人数据
- **违反标准**：GB/T 35273-2020 用户权利保障
- **解决方案**：在个人信息页面添加"注销账号并删除数据"功能
- **预计工时**：2小时

### P1 - 中优先级（建议修复）

#### 4. [ ] 收紧 CORS 配置
- **文件**：`backend/config/settings.py`
- **问题**：`BACKEND_CORS_ORIGINS: ["*"]` 过于宽松
- **解决方案**：
  - 开发环境：允许 localhost
  - 生产环境：仅允许指定域名
- **预计工时**：30分钟

#### 5. [ ] 添加标准引用注释
- **文件**：
  - `backend/validators/health_validator.py`
  - `frontend/health_assessment_system/config/indicator_reference.json`
- **问题**：代码中未标注参考的国家/行业标准
- **解决方案**：在健康数据参考值定义处添加标准引用注释
- **预计工时**：1小时

#### 6. [ ] 生产环境 HTTPS 配置
- **文件**：部署配置文件
- **问题**：需确保生产环境强制使用 HTTPS
- **违反标准**：GB/T 35273-2020 数据传输安全
- **解决方案**：配置 SSL 证书，强制 HTTPS 跳转
- **预计工时**：1小时

---

### P2 - 低优先级（优化项）

#### 7. [ ] 适老化对比度检查
- **文件**：全局样式文件
- **问题**：部分文字颜色（如 `text-slate-400`、`text-slate-500`）对比度可能不足
- **参考标准**：GB/T 37668-2019 无障碍可访问性要求（WCAG 2.1 AA）
- **解决方案**：
  - 使用对比度检测工具验证
  - 将浅色文字调整为更深的颜色
  - 确保对比度 ≥ 4.5:1（普通文字）或 ≥ 3:1（大文字）
- **预计工时**：1-2小时

#### 8. [ ] 焦点指示器增强
- **文件**：`frontend/src/index.css` 或组件样式
- **问题**：键盘导航时焦点状态不够明显
- **参考标准**：GB/T 37668-2019
- **解决方案**：增强 focus 样式，使用更明显的边框或背景色
- **预计工时**：30分钟
